# ArayaShiki Ecosystem Narrative

## Purpose

This document defines how `ArayaShiki Lab` should be presented when the repository is no longer framed as a single-product pilot only.

Use it for:

- landing page messaging
- company overview documents
- solution packaging
- partner and customer-facing summaries

## Positioning

`ArayaShiki Lab` builds a smart hatchery and aquaculture ecosystem for farms, hatcheries, research institutes, and private operators.

The company does not position itself as a sensor vendor only, a dashboard only, or a single-purpose smart farm product.

The core narrative is:

1. real operations start with usable data capture
2. the same platform must accept both human input and board input
3. monitoring is only the first layer
4. the long-term value is controlled automation and process orchestration

## Company Promise

ArayaShiki Lab connects:

- manual operational records from people
- telemetry from sensors and boards
- rule-based actions
- future process automation inside hatchery and farm workflows

The system must support different customers without forcing them to adopt the same operating model on day one.

## Primary Audiences

### Research And Education

Need:

- structured data collection
- repeatable observation workflows
- controlled experiments
- a bridge between field activity and data history

### Farm And Hatchery Operators

Need:

- daily records that fit real work
- monitoring that works with limited staff
- better visibility into water, process, and task status
- a practical path toward automation

### Private Sector And Enterprise Operators

Need:

- standard operating visibility
- permissioned data access
- board and control integrations
- configurable automation and reporting

## Ecosystem Definition

The ecosystem has four layers.

### 1. Human Data Capture

The platform must support structured records entered by people during real operations.

Examples:

- daily hatchery work logs
- preparation records
- mortality or nursery records
- research observations

### 2. Device And Board Data

The platform must accept data from hardware, sensors, and boards.

Examples:

- water quality readings
- environmental telemetry
- device status
- board-originated events

### 3. Rule-Based Automation

The platform must support automation such as:

- if a threshold is crossed, send an alert
- if a condition is met, switch a device on or off
- if a state persists, escalate or notify

### 4. Process Automation

The long-term platform vision includes multi-step control logic.

Example:

- produce water at `25 ppt`
- read salinity
- calculate required freshwater and saltwater input
- control the relevant pumps
- pass water into the next tank or filtration stage
- confirm completion and store the result as traceable operational data

## Flagship Product

`Daily Complete` is the first flagship solution inside the ArayaShiki ecosystem.

It should be framed as:

- the first operational surface
- the first production-ready workflow layer
- the point where manual data entry, operational visibility, and future automation converge

Do not frame it as the entire company.

Use phrasing such as:

- `Daily Complete is the flagship operating layer of the ArayaShiki ecosystem.`
- `Daily Complete starts with daily operations and expands toward connected automation.`

## Capability Status

Current status:

- monitoring exists at the `SB-00` level
- broader board and actuator coverage is planned, not yet fully delivered

Messaging rule:

- describe current monitoring as available now
- describe broader automation as roadmap or planned capability
- do not imply that full closed-loop automation is already deployed

## Packaging Direction

Internally, ArayaShiki Lab may continue building a broad feature set first.

Externally, the ecosystem should be presented as packages or solution tracks so each audience can quickly understand what is relevant to them.

Recommended outward packaging:

1. `Research & Education`
2. `Farm Operations`
3. `Enterprise & Custom Automation`

This keeps the platform broad without making the market-facing story feel vague.

## Messaging Rules

### Say

- smart hatchery and aquaculture ecosystem
- connected records, monitoring, and automation
- built for farms, hatcheries, and applied research
- starts with operational data and grows toward automation

### Avoid

- presenting ArayaShiki Lab as only a single-device product
- implying every automation capability is already shipped
- speaking only in generic smart farm language with no hatchery specificity
- overclaiming AI or autonomy before the control layer is proven

## Landing Page Structure

The homepage should be company-led.

Recommended order:

1. `ArayaShiki Lab` hero
2. ecosystem explanation
3. flagship solution: `Daily Complete`
4. audience-specific solution tracks
5. automation roadmap
6. call to action

## Core One-Line Summary

`ArayaShiki Lab builds a smart hatchery and aquaculture ecosystem that connects human records, board data, monitoring, and future automation in one operational platform.`
